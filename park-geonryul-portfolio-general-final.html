PROJECT 01 · DATA SERVICE

ER-Dodge

게임 데이터를 분석해 사용자가 다음 경기에서 취할 행동을 제안하는 웹서비스

외부 게임 데이터를 수집·정규화하고 사용자 지표와 비교해 결과를 반환합니다. 기준 통계가 일시적으로 불안정해도 서비스가 멈추지 않도록 자동 갱신, 캐시, 폴백 구조를 적용했습니다.

Node.jsNetlify FunctionsGitHub ActionsExternal API

담당 및 설계

외부 데이터 수집과 화면용 JSON 구조 설계
6시간 주기 기준 데이터 자동 갱신
원격 데이터 실패 시 캐시·번들 데이터로 대체
AI 피드백 기능을 구조화된 응답 형태로 확장

문제와 해결

외부 기준 데이터가 불안정할 때 서비스 전체가 멈추지 않도록 단계별 폴백 구조를 적용했습니다.
분석 결과가 단순 수치 설명에 그치지 않도록 사용자가 다음 경기에서 실행할 행동 중심으로 출력 구조를 개선했습니다.

PROJECT 02 · DEPLOYMENT SYSTEM

Student Progress Kiosk

학교 키오스크 PC에서 학생별 프로젝트 진행상황을 조회·관리하는 현장형 시스템

공개 키오스크와 관리자 화면이 같은 데이터베이스를 사용하도록 구성하고, 신규 PC에서도 같은 환경으로 실행할 수 있도록 애플리케이션과 DB를 컨테이너화했습니다.

Node.jsMySQLDocker ComposeNginx

담당 및 설계

학생·프로젝트·진행단계·피드백 데이터 구조 설계
조회 화면과 관리자 CRUD 기능 구현
App·DB·Nginx를 Docker Compose로 구성
포트·환경변수·볼륨·서비스 의존성 설정

문제와 해결

PC마다 설치 환경이 달라지는 문제를 줄이기 위해 앱·DB·프록시를 하나의 Compose 구성으로 통합했습니다.
DB 초기화와 서비스 시작 순서에서 발생한 오류를 헬스체크와 의존성 설정으로 정리했습니다.

PROJECT 03 · PUBLIC DATA

Assembly Tracker

국회 공개 API를 통합해 의원과 법안 활동을 탐색하는 데이터 대시보드

서로 다른 공공 API의 응답 형식을 공통 데이터 구조로 정리하고, 검색·필터·차트에서 동일한 기준을 사용할 수 있도록 가공했습니다.

JavaScriptPublic APIData NormalizationCharts

담당 및 설계

여러 공개 API를 하나의 사용자 흐름으로 통합
서버 경유 방식으로 인증키 노출 방지
의원·법안 응답을 화면용 공통 구조로 변환
검색·필터·통계 시각화 구현

문제와 해결

API마다 다른 필드명과 응답 형식을 공통 모델로 정규화해 화면 로직의 중복을 줄였습니다.
인증키를 브라우저에 노출하지 않도록 서버를 경유하는 요청 구조로 변경했습니다.
